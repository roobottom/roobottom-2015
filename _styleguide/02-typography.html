--- title: Typography ---

This site is heavily text based, and utilises lots of standard elements. To protect the rest of the site from overzealous text styles, all blocks of text are wrapped in a scope class .s-prose.

Second level heading

Second level headings h2 are used in a post. Multiple second level headings can be used to introduce separate ideas in a post.

Third level heading

Third level headings h3 are used in a post as a sub level heading to a h2.

Fourth level heading

Fourth level headings h4 are used in a post as a sub level heading to a h3.

Horizontal rule

The horizontal rule is used to separate one concept from another in a post.

Blockquotes

The blockquote element is used when quoting from another source

The history of the Olympics is not only one of remarkable sporting achievement, but acts of incredible artistry too.

Ordered lists

Ordered lists ol are used, as their name suggests, to list items which have an order. They can be nested.

  1. UX tasks
    1. User research
    2. Creating personas
    3. Story writing
    4. Sketching
    5. Wireframes
  2. Design tasks
    1. Sketching
    2. Screen mockups
    3. Pattern systems
    4. Screen flows

Unordered lists

Unordered lists ul are used, as their name suggests, to list items which don't have an order. They can be nested.

  • Colours
    • Red
    • Yellow
    • Pink
    • Green
  • Textures
    • Rough
    • Sand
    • Grass
    • Smooth

Links

The a tag is used to link text to another location on the web.
This is an example of a link.

Emphasis

The em tag is used to add emphasis to a word or phrase.

Importance

The strong tag is used to add importance to a word or phrase.

Abbreviations

The abbr tag is used along with it's title attribute to explain any abbreviations used, such as HTML or TL;DR

Dates and time

The time tag is used to display a date or time. For example, this page was written on Monday, 10th August 2015.

Code

For snippets of code inline in text the code tag is used, for example typing ls -a into terminal will show you all files including the . files

However, if there's more code to be shown then multiline code can be wrapped in pre tags and then styled automagically with the Rouge Highlighter, like this: {% highlight xslt linenos %}{% raw %}

{% endraw %}{% endhighlight %}
